# Дисертаційна робота: Метод перетворення опису програми у код інструментальною мовою програмування

## Мета

Розробка методу та інструменту автоматизованого перетворення формального опису програмної моделі (на основі предикатів і темпоральної логіки) у коректний код цільової мови програмування (C++).

Ключова ідея: опис моделі → НСА (структура Кріпке) → проміжне подання (IR) → цільова програма.

---

## Теоретична база

### Структура Кріпке

Математична модель програми: **M = (S, S₀, R, AP, L)**

| Символ | Значення |
|--------|----------|
| S      | Множина станів моделі |
| S₀     | Множина початкових станів |
| R = S × S | Множина зв'язків між станами |
| AP     | Кінцева множина предикатів |
| L = 2^AP | Кількість можливих сполучень предикатів |

### Формат предикату

```
{умова} [перелік дій]
```

- `{умова}` — логічна частина (функція переходу); може містити операції >, <, >=, <=, =, ≠, AND, OR, NOT, а також темпоральні оператори pUq та Xr
- `[дії]` — виконавча частина (функція збудження); арифметичні вирази, присвоєння, read/print

### Розширення синтаксису (реалізовано в інструменті)

```
@thread {умова} [дії]           — потік (паралельна гілка)
:label  {умова} [дії]           — мітка для переходу goto
        {умова} [goto label;]   — безумовний перехід
        {умова} [дії] <sem: res> — семафорний примітив (таблиця MEMO)
```

### Внутрішнє подання (база даних)

**Таблиця MAIN** — стани автомату:

| Поле | Тип | Опис |
|------|-----|------|
| Id_state | INT | Ідентифікатор стану |
| Type | VARCHAR | initial / normal / final |
| List_of_actions | MEMO | Список дій стану |
| Mark | BOOL | Прапор відвідування (захист від зациклення) |

**Таблиця CONNECT** — переходи між станами:

| Поле | Тип | Опис |
|------|-----|------|
| Id_state (current) | INT | Поточний стан |
| Condition | MEMO | Умова переходу |
| Next_Id_state | INT | Наступний стан |

**Таблиця MEMO** — семафорний примітив:

| Поле | Тип | Опис |
|------|-----|------|
| Id_state (current) | INT | Стан, що займає ресурс |
| Sem | VARCHAR | Назва семафору |
| Resource | VARCHAR | Назва підпрограми-ресурсу |

## Стековий алгоритм

Центральний механізм системи — розбір виразів і розпізнавання патернів у моделі.

**Принцип:** лексеми розбиваються на пари `<лексема даних, лексема дії>`, причому лексеми дії мають пріоритет.

```
<лексема даних> := <змінна> | <константа>
<лексема дії>   := <оператор (+, -, *, /)> | <ліва дужка>
```

**Пріоритети операторів:**

| Оператор | Пріоритет у стеку | Пріоритет входу |
|----------|-------------------|-----------------|
| +, -     | 1                 | 1               |
| *, /     | 2                 | 2               |
| (        | 0                 | 3               |
| унарний +/- | 3            | 4               |

**Приклад:** `A * (B + C / D) - E`

1. A * → поточна пара
2. ( → стек (пріоритет входу=3, порівняння=0)
3. B + → A* у стек, B+ у поточну пару
4. C / → ( у стек, B+ у попередню, C/ у поточну
5. D ) → обробка C/D=t1, виштовхуємо B+ → B+t1=t2, знищуємо (, виштовхуємо A* → A*t2=t3
6. E → обробка t3-E

## Патерни у проміжному поданні (IR)

| Вузол IR | Конструкція | Опис |
|----------|-------------|------|
| `EX`     | лінійний блок | Послідовні присвоєння |
| `IF1`    | if/else (goto-based) | Два виходи з поточного стану |
| `IF2`    | switch/case | Розгалуження за значеннями одного параметру |
| `IF3`    | if / else if | Ланцюжок умов (кілька послідовних умовних переходів) |
| `DO1`    | for (з перерахунком) | Цикл n разів |
| `DO2`    | while (з передумовою) | Самоперехід (петля) у стані |
| `DO3`    | do-while (з післяумовою) | Наступний стан повертається до поточного |
| `THREAD` | паралельна гілка | `@threadName` — окремий потік |
| `RETURN` | завершення | Кінцевий стан без виходів |

### Приклад розпізнавання патернів (алгоритм сортування бульбашкою)

```
{TRUE} [MAS(N) of symb, N, J1=N-1, I1=N-1, J=1, I=1: integer, temp: symb]
({J1≠0} ([I=1 {I≤I1} [{MAS(I)>MAS(I+1)} [MAS(I)=temp, MAS(I)=MAS(I+1), MAS(I+1)=temp]
          OR {MAS(I)≤MAS(I+1)} [empty]])
X {true} [I=I+1, {I≤I1} [loop]]) U {I>I1} [J1=J1-1, I1=J1, {J1≠0} [loop]])
OR {J1=0} [stop]
```

Проміжне подання:
```
CYC1: CYC2: IF MAS(I)>MAS(I+1), swap, I=I+1, IF I≤I1 JMP CYC2
OTHERWISE J1=J1-1, I1=J1, IF J1≠0 JMP CYC1 OTHERWISE STOP
```

## Наступні кроки

### 1. Система типів даних
На першому етапі необхідно визначити всі дані та їх типи. Поточний генератор оголошує всі змінні як `int`. Потрібно:
- розпізнавати тип з першого стану (`{TRUE}[MAS(N) of symb, N: integer, temp: symb]`)
- підтримувати: `integer`, `char`/`symb`, `float`, `string`, масиви

### 2. Патерни IF2 та DO1
- **IF2**: розпізнавати послідовність переходів з однаковим ідентифікатором та різними значеннями → генерувати `switch/case`
- **DO1**: розпізнавати патерн підготовки + цикл n разів → генерувати `for`

### 3. Темпоральні оператори
Повна підтримка `pUq` (Until) і `Xr` (Next) у синтаксисі вхідного опису — ключова теоретична частина, що наразі не відображена в парсері.

### 4. Шаблони/макроси
Подібно до макросів в асемблері — попередньо визначені шаблони (сортування, пошук тощо), що можна підключити до моделі.
